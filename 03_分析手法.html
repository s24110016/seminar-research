3.1 手法の選定

本研究では、以下の2手法を比較する。ベースラインとして線形回帰を用い、非線形な関係を捉える手法として回帰木（rpart）(James ほか 2023年) を採用する。

3.2 線形回帰（ベースライン）

線形回帰モデルは、目的変数 \(y\) を説明変数 \(\mathbf{x} = (x_1, x_2, \ldots, x_p)\) の線形結合で表現する。

\[ y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots + \beta_p x_p + \varepsilon \tag{3.1}\]

ここで \(\beta_0\) は切片、\(\beta_1, \ldots, \beta_p\) は回帰係数、\(\varepsilon\) は誤差項である (James ほか 2023年)。

3.3 回帰木（rpart）

回帰木は、説明変数の空間を再帰的に分割し、各領域ごとに目的変数の平均値を予測するノンパラメトリック手法である (James ほか 2023年)。R の rpart パッケージはR に標準搭載されており、追加インストール不要で利用できる。

James, Gareth, Daniela Witten, Trevor Hastie, Robert Tibshirani, と Jonathan Taylor. 2023年. An Introduction to Statistical Learning: with Applications in R. 2nd 版. Springer. https://doi.org/10.1007/978-3-031-38747-0.

分割の基準

各ノードにおいて、残差二乗和（RSS）を最小化する分割点を探索する。

\[ \text{RSS} = \sum_{x_i \in R_1} (y_i - \hat{y}_{R_1})^2 + \sum_{x_i \in R_2} (y_i - \hat{y}_{R_2})^2 \tag{3.2}\]

ここで \(R_1, R_2\) は分割後の2つの領域、\(\hat{y}_{R_k}\) は各領域内の目的変数の平均値である。

決定木の構造

剪定（Pruning）

回帰木は深くなるほど訓練データに過学習する。これを防ぐため、コスト複雑度剪定（cost-complexity pruning）を行う。

\[ \text{Cost} = \text{RSS} + \alpha \cdot |T| \tag{3.3}\]

ここで \(|T|\) は木の終端ノード数、\(\alpha\) は複雑度パラメータ（cp）である。\(\alpha\) が大きいほど単純な木が選ばれる。

3.6 精度評価指標

モデルの予測精度を評価するため、以下の3指標を使用する。

平均絶対誤差（MAE）

\[ \text{MAE} = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i| \tag{3.4}\]

二乗平均平方根誤差（RMSE）

\[ \text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2} \tag{3.5}\]

決定係数（R²）

\[ R^2 = 1 - \frac{\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}{\sum_{i=1}^{n}(y_i - \bar{y})^2} \tag{3.6}\]

ここで \(y_i\) は実測値、\(\hat{y}_i\) は予測値、\(\bar{y}\) は実測値の平均、\(n\) はサンプル数である。

式 3.4 は外れ値に頑健な指標、式 3.5 は大きな誤差をより重く評価する指標、式 3.6 はモデルの説明力を0〜1で表す指標である。

3.7 追加パッケージの利用（発展）

本テンプレートはR の基本パッケージのみで動作するが、研究の目的に応じてより高度なパッケージを導入できる。

重要パッケージの追加手順
  1. R コンソールで install.packages("パッケージ名") を実行（初回のみ）
  2. .qmd ファイルのコードブロック内で library(パッケージ名) を記述
  3. quarto render で正常にビルドできることを確認

代表的な追加パッケージ

パッケージ 用途 インストール
tidyverse データ操作・可視化の統合環境 install.packages("tidyverse")
randomForest ランダムフォレスト install.packages("randomForest")
lightgbm 勾配ブースティング（LightGBM） install.packages("lightgbm")
plotly インタラクティブなグラフ install.packages("plotly")

使用例：LightGBM で回帰木を置き換える場合

# 1. インストール（R コンソールで一度だけ実行）
install.packages("lightgbm")

# 2. .qmd ファイル内で読み込んで使用
library(lightgbm)

dtrain <- lgb.Dataset(
  data  = as.matrix(train_data[, -which(names(train_data) == "count")]),
  label = train_data$count
)

params <- list(objective = "regression", metric = "rmse",
               num_leaves = 31, learning_rate = 0.1)

lgbm_fit <- lgb.train(params, dtrain, nrounds = 500)
警告注意
  • 追加パッケージを使う場合、quarto render 時にそのパッケージがインストール済みである必要がある
  • エラーが出たら、まず install.packages("パッケージ名") を実行してから再度 quarto render する
  • GitHub Pages にデプロイする際、ビルドは自分の PC 上で行われるため、自分の PC にパッケージがあれば問題ない
